# Contributing to DriftCode Auditor

Thanks for your interest in improving DriftCode Auditor!

We especially welcome contributions from developers who use AI coding tools daily (Cursor, Claude, Copilot, etc.) and want to help catch more real-world mistakes in generated code.

## How to Contribute

### Reporting Issues
- Use the appropriate issue template when opening a new issue.
- For "What the tool caught for me" success stories, use the dedicated template.

### Suggesting New Rules
- Open a feature request or use the Discussions tab.
- Describe the AI-generated mistake pattern you want to catch and (if possible) provide a small example.

### Code Contributions
- Fork the repo and create a feature branch.
- Make sure existing tests still pass: `python -m pytest tests/`
- Keep changes focused and well-documented.

### Good First Issues
Look for issues labeled `good first issue` or `help wanted`.

## Communication
- GitHub Discussions is the best place for ideas, questions, and feedback.
- We keep the project friendly and focused on practical quality improvements for AI-assisted code.

Thank you for helping make AI-generated code more reliable!
